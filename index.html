Core Capabilities

Everything You Need,
Nothing You Don't

💬

Real-Time Chat

Instant message delivery over persistent TCP connections. UTF-8 encoded, MSG:: protocol.

MSG:: protocol
📁

Chunked File Transfer

Files sent as 16KB binary chunks — exactly like the C repo's BUF_SIZE=16*1024. Receiver reconstructs and verifies integrity.

FILE:: protocol
🔁

Go-Back-N Reliability

GBN sliding window protocol — ACK/NACK handling, retransmission, configurable window size N.

GBN · Window=16
🏠

Chatrooms

Create or join named chatrooms — public or password-protected. Broadcast @all or unicast @username.

create / join
🧵

Threaded I/O

Concurrent send/receive threads for full-duplex communication. No operation blocks another.

Multi-threaded
🔒

Fully Offline

Operates entirely on your local WiFi. No cloud, no internet. Bind to port 9090 and communicate within LAN.

LAN Only
Workflow

Up and Running in
Three Simple Steps

01
🖥️

Host Starts Server

One device binds to port 9090 and listens for incoming TCP connections on the local network.

# server.py
server.bind(('0.0.0.0', 9090))
server.listen(25)
→
02
🔗

Peer Connects

Client enters username, creates or joins a chatroom. Server verifies uniqueness and routes messages.

# client.py
client.connect(('192.168.1.10', 9090))
→
03
🚀

Chat & Transfer

Send messages or request files. File transfers use UDP + Go-Back-N. Chat uses TCP relay.

Under the Hood

Technical Architecture

🐍

Python / C

server.py, client.py, transfer.c — socket programming with built-in networking libraries

🔌

TCP Sockets

Reliable, ordered, connection-oriented data transmission for chat and control messages

📡

UDP + Go-Back-N

File transfers use UDP with GBN sliding window for performance and reliability

🧵

Threading

thread.start_new_thread() for concurrent send/receive — full-duplex I/O

📦

16KB Chunks

BUF_SIZE = 16*1024 — memory efficient, smooth progress tracking

🏠

Chatrooms

Named rooms with optional password protection, broadcast and unicast messaging
